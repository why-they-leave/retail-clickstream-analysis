# Rule 03 — 분석 워크플로우

## 표준 분석 순서
모든 데이터 분석은 아래 단계를 기본으로 따른다:

1. **데이터 로드 & 기본 검사**
   - shape, dtypes, null count, duplicates
2. **EDA (탐색적 데이터 분석)**
   - 분포 확인, 이상치 탐지, 상관관계
3. **전처리**
   - 결측치 처리, 타입 변환, 인코딩
4. **분석/모델링**
   - 목적에 따른 통계 검정 또는 ML 적용
5. **결과 해석 & 시각화**
   - 핵심 인사이트 3가지 이내로 요약
6. **다음 단계 제안**
   - 추가 분석 방향 또는 개선 사항

## 단계 건너뛰기 금지
- 사용자가 특정 단계를 요청해도 선행 단계 결과 없이 진행하면 경고한다
- 예: EDA 없이 바로 모델링 요청 시 → EDA 먼저 제안

## 재현성 확보
- 모든 분석에 `random_state=42` 또는 `np.random.seed(42)` 포함
- 분석 날짜를 코드 상단 주석으로 기록 권장

## 결과 저장
- 중간 결과는 `results/` 폴더에 저장하는 패턴 권장
- 최종 리포트는 Markdown 또는 HTML 형식 제안
