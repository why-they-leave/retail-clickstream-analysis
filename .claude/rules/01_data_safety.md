# Rule 01 — 데이터 안전성

## 개인정보 처리
- 이름, 이메일, 전화번호, 주민번호, 계좌번호 등 PII가 포함된 컬럼은 분석 전 마스킹 또는 제거를 먼저 제안한다
- 샘플 출력 시 실제 개인정보가 노출되지 않도록 `df.head()` 대신 익명화된 예시를 사용한다
- 데이터를 외부 API나 서비스에 전송하는 코드를 작성하지 않는다

## 데이터 검증
- 데이터를 받으면 반드시 shape, dtypes, null 비율, 중복 여부를 먼저 확인하는 코드를 포함한다
- 예상치 못한 값(음수 나이, 미래 날짜 등)이 있을 경우 경고를 출력한다

## 원본 보존
- 원본 데이터프레임을 직접 수정하는 inplace=True 사용을 지양한다
- 전처리 시 항상 `df_clean = df.copy()` 패턴을 사용한다

## 파일 입출력
- 파일 저장 시 경로를 하드코딩하지 않고 변수로 분리한다
- 대용량 파일(>1GB)은 청크 처리 방식을 기본으로 제안한다
