## 1. 분석 목적

GPLR(Graph-based Persona Label propagation via Random walk) 알고리즘을 퍼널 데이터셋에 적용하여 16,268명 고객에게 소비자 페르소나를 부여하고, 페르소나별 퍼널 전환율을 비교 분석한다.

전체 유저를 LLM으로 직접 라벨링하는 대신 일부(2.5%)만 라벨링한 뒤 Random Walk로 전파하여 비용과 확장성을 확보한다.

## 3. 파이프라인 구현 현황

### Step 1 — 데이터 전처리 ✅

`src/datasets/preprocess_funnel.py`

퍼널 4개 테이블을 GPLR이 요구하는 MBA 포맷(세미콜론 구분 CSV)으로 변환한다.

```
orders + order_items + products → data/funnel_mba_format.csv

컬럼 매핑:
  customer_id    → CustomerID
  product name   → Itemname
  quantity       → Quantity
  order_id       → BillNo
  order_time     → Date
  unit_price_usd → Price
  country        → Country
```

events 테이블에 customer_id가 없어 `events.session_id → sessions.customer_id` 조인이 선행된다.

### Step 2 — 페르소나 목록 생성 ✅

`src/persona/generation.py` | LLM: Upstage Solar Pro

논문 Appendix B의 3단계 프로세스로 퍼널 데이터셋 특화 20개 페르소나를 생성한다.

```
Step 2-1 (40회, 병렬): 랜덤 100명 구매이력 → LLM → 20개 후보 × 40세트 = 800개
Step 2-2 ( 8회, 병렬): 5세트(100개) 랜덤 샘플링 → LLM → 정제된 20개 × 8세트 = 160개
Step 2-3 ( 1회):       8세트(160개) → LLM → 최종 20개 확정
```

### Step 3 — 페르소나 라벨링 (Prototype User 선정) ✅

`src/persona/labeling.py` | LLM: Upstage Solar Pro

**유저 라벨링**

- 전체 16,268명 중 **2.5% (407명) 랜덤 샘플링** → 각 유저의 train 구매이력 기반으로 LLM 라벨 부여
- Train/Test 분할: 유저별 구매이력 **80/20 랜덤 분할** (`random.seed(191)`)

| 항목 | 수치 |
|------|------|
| 샘플 유저 | 407명 |
| 라벨 성공 (prototype users) | **114명 (28.0%)** |
| Unrepresentable / 실패 | 293명 (72.0%) |

**상품 라벨링**

- 전체 1,195개 상품 전수 라벨링

| 항목 | 수치 |
|------|------|
| 라벨된 상품 | **244개 / 1,195개 (20.4%)** |

**페르소나별 유저 분포 (상위 5개)**

### Step 4 — GPLR Random Walk ⬜ 미완

`src/random_walker/ReverseRW.py` (RevAff 알고리즘 구현 존재)

prototype users 114명을 시드로 Reverse Affinity Random Walk를 수행하여 나머지 16,154명에게 페르소나를 전파한다. 구현체(`ReverseRW.py`, `PageRank.py`)는 존재하며, 퍼널 데이터셋에 연결하는 실행 스크립트가 필요하다.

## 4. 방법론적 고려사항

### 라벨 실패율 72% 원인

유저의 약 45%가 train 아이템이 1개뿐이라 LLM이 구매 패턴을 특정할 수 없어 `Unrepresentable` 처리된다. 이는 Step 4 Random Walk 단계에서 주변 유저 페르소나로 채워질 예정이므로 치명적 문제는 아니다.

### 합성 데이터 아티팩트

상품명이 `"SSD MediumBlue 149"` 형태(카테고리+색상코드+ID)로 구성되어 있어 `Color-Conscious Buyer` 같은 비현실적 페르소나가 생성됐다. 실제 데이터 적용 시 미발생이 예상된다.

### 페르소나 중복 및 세분화

- **중복**: #17 E-book Devourer, #18 Frequent Book Buyer → #1 Bookworm과 중복 가능성
- **세분화 과다**: #10~#14가 상위 카테고리(Beauty, Toy, Apparel)의 서브타입으로 과세분화됨

현재는 그대로 유지하되 Step 4 완료 후 실제 분포 확인 뒤 통합 여부를 결정한다.
