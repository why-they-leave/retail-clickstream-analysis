## 5. 카테고리 다양성 및 집중도

`top_view_category`, `top_purchase_category`만으로는 고객이 한 카테고리에 집중하는지, 여러 카테고리를 넓게 보는지 알기 어렵다. 이를 보완하기 위해 category diversity와 dominant category ratio를 추가 확인했다.

| 피처 | mean | median | 해석 |
|------|------|--------|------|
| `category_diversity_view` | 6.13 | 7 | 고객이 조회한 서로 다른 카테고리 수 |
| `dominant_view_category_ratio` | 0.345 | 0.319 | 전체 조회 중 최다 조회 카테고리 비중 |
| `category_diversity_purchase` | 2.43 | 2 | 고객이 구매한 서로 다른 카테고리 수 |
| `dominant_purchase_category_ratio` | 0.663 | 0.600 | 전체 구매 중 최다 구매 카테고리 비중 |

**핵심**:

- 고객은 평균 6개 이상의 카테고리를 조회한다. 조회 카테고리 다양성은 거의 포화되어 있어 단독 세그먼트 피처로는 구분력이 약할 수 있다.
- 반면 구매 카테고리 다양성은 평균 2.4개로 훨씬 좁다.
- 구매 top category 집중도는 평균 66.3%로 높다.

즉 이 쇼핑몰의 고객은 **탐색은 넓게 하지만 구매는 일부 카테고리에 집중하는 경향**이 있다.

## 6. 후보 피처 상관관계

상관관계 분석 결과, 일부 후보 피처는 같은 행동 축을 중복해서 담고 있다.

| 피처 pair | corr | 판단 |
|-----------|------|------|
| `total_spend_log` - `avg_order_value_log` | 0.93 | 구매 금액 계열 중복. 둘 다 넣기보다 하나를 대표로 선택 |
| `purchase_per_session` - `purchase_per_view` | 0.89 | 구매 전환 비율 계열 중복 |
| `session_count` - `page_view_count` | 0.89 | 활동량 계열 중복 |
| `is_purchaser` - `total_spend_log` | 0.88 | 미구매 고객의 spend=0 영향 |
| `category_diversity_purchase` - `dominant_purchase_category_ratio` | -0.82 | 구매 다양성과 구매 집중도의 반대 표현 |
| `page_view_count` - `add_to_cart_count` | 0.81 | 조회량과 장바구니 수가 강하게 연결 |

**해석**:

- 세그먼트 입력에 후보 피처를 모두 넣으면 활동량/구매량 축이 과하게 반영될 수 있다.
- `atc_rate`는 `session_count`, `page_view_count`와 거의 독립적이어서 조회 대비 장바구니 전환 성향을 나타내는 좋은 후보 피처다.
- `purchase_per_view`는 `purchase_per_session`과 중복되고 단위 해석이 애매하므로 보류한다.
- `category_diversity_purchase`와 `dominant_purchase_category_ratio`는 둘 중 하나를 세그먼트 입력으로 선택하고, 다른 하나는 summary용으로 남기는 것이 적절하다.

## 7. 운영형 Window 피처 분포

1차 v2 실험은 기존 전체 기간 누적 피처를 사용하되, 운영 반영 시 필요한 최근 30일/90일/365일 window 피처의 분포도 확인했다.

### 활성 고객 비율

| window | 세션 있음 | 조회 있음 | 장바구니 있음 | 구매 있음 |
|--------|-----------|-----------|---------------|-----------|
| 30d | 8.05% | 8.05% | 5.55% | 2.32% |
| 90d | 22.87% | 22.87% | 16.45% | 7.15% |
| 365d | 64.05% | 64.05% | 50.06% | 25.42% |

### 주요 window 피처 평균

| 피처 | 30d | 90d | 365d |
|------|-----|-----|------|
| `page_view_count` | 0.38 | 1.17 | 4.64 |
| `order_count` | 0.024 | 0.074 | 0.295 |
| `total_spend_log` | 0.10 | 0.32 | 1.16 |
| `atc_rate` | 0.266 | 0.271 | 0.266 |

**해석**:

- 최근 30일/90일 window는 전체 고객 기준으로 sparse하다.
- 최근 365일 window는 활동 고객 64%, 구매 고객 25.4%로 운영형 피처 후보로 더 안정적이다.
- Full과 US의 window 피처 평균 차이도 거의 없어, US-only는 전체 고객의 축소 샘플처럼 동작한다.

## 8. 합성 이커머스 쇼핑몰 고객/로그 특성 요약

- **전체 vs US 차이 없음**: 모든 수치형 피처에서 전체와 US 평균이 거의 동일. 페르소나 생성 시 전체/US 비교의 의미가 제한적일 수 있음.
- **미구매 고객 18.7%**: 세션/이벤트는 있으나 주문 없는 고객이 상당수 존재. 퍼널 이탈 분석의 핵심 대상.
- **Books · Beauty 카테고리 집중**: 조회와 구매 모두 Books, Beauty가 1·2위. 이 두 카테고리 중심 페르소나가 다수를 차지할 가능성 높음.
- **조회-구매 불일치 60%**: 고객이 가장 많이 본 카테고리와 실제 구매 카테고리가 다른 경우가 더 많아, 조회 행동만으로 구매 의도를 예측하는 데 한계가 있음.
- **넓은 탐색, 좁은 구매**: 평균 조회 카테고리는 6개 이상이지만 평균 구매 카테고리는 2.4개 수준. 고객은 여러 카테고리를 둘러본 뒤 일부 카테고리에 집중 구매한다.
- **30/90일 최근 로그는 sparse**: 운영형 피처로는 365일 window가 더 안정적이며, 30/90일 피처는 최근 활성 고객 전용 보조 피처로 보는 것이 적절하다.

### LLM 입력 방식

v2에서는 LLM이 개별 고객의 상품명을 보고 페르소나를 직접 생성하지 않는다. 고객별 피처 계산과 segment assignment는 코드에서 먼저 수행하고, LLM은 segment 단위 요약 통계를 보고 이름과 설명을 붙이는 역할로 제한한다.

흐름은 아래와 같다.

```text
고객별 로그/주문 데이터
→ 고객별 수치/범주형 피처 계산
→ clustering 기반 segment_id 부여
→ segment_id별 요약 통계 생성
→ LLM이 segment name / description / evidence 생성
```

LLM에는 개별 고객 row가 아니라 아래와 같은 segment summary를 입력한다.

| 그룹 | LLM 입력 후보 |
|------|---------------|
| Segment 규모 | `customer_count`, `customer_ratio` |
| 활동량 | `avg_session_count`, `avg_page_view_count`, `avg_add_to_cart_count` |
| 의도/전환 | `avg_atc_rate`, `purchaser_ratio`, `avg_order_count`, `avg_purchase_per_session` |
| 구매 규모 | `avg_total_spend_log`, `median_total_spend`, `avg_avg_order_value_log` |
| 최근성 | `avg_recency_session_days`, `avg_recency_order_days` |
| 카테고리 패턴 | `top_view_categories`, `top_purchase_categories` |
| 카테고리 집중/일치 | `avg_category_diversity_purchase`, `avg_dominant_purchase_category_ratio`, `view_purchase_category_match_rate` |

예시 입력:

```text
Segment 3 summary:

- Customers: 412
- Avg page views: 41.5
- Avg add-to-cart rate: 31.0%
- Purchaser ratio: 22.0%
- Avg order count: 0.4
- Avg recency session days: 38
- Top view categories: Fashion 38%, Beauty 24%, Sports 17%
- Top purchase categories: Fashion 41%, Beauty 21%
- Avg purchase category diversity: 1.7
- Avg dominant purchase category ratio: 0.72
- View-purchase category match rate: 68%

Task:
Name this segment and describe it using only the evidence above.
Do not infer income, family status, job, lifestyle, age, or personality traits
that are not supported by the data.
Return JSON with segment_name, description, and evidence.
```

기대 출력은 아래처럼 수치 근거를 포함한 요약형 라벨이다.

```json
{
  "segment_name": "High-Intent Fashion and Beauty Browsers",
  "description": "Customers in this segment frequently browse Fashion and Beauty products, add items to cart at a relatively high rate, but complete purchases less often than their browsing behavior suggests.",
  "evidence": [
    "avg_atc_rate is 31.0%",
    "purchaser_ratio is 22.0%",
    "Fashion and Beauty are the dominant view and purchase categories",
    "avg_dominant_purchase_category_ratio is 0.72"
  ]
}
```

이 구조의 목적은 LLM이 synthetic 상품명에서 과잉 추론하지 않도록 막고, 이미 계산된 세그먼트의 행동적 특징을 사람이 읽기 쉬운 이름과 설명으로 번역하게 하는 것이다. 즉 raw 상품명으로 직업·소득·가족상태·생활양식을 추론하지 않으며, 카테고리 정보는 `top_view_categories`, `top_purchase_categories`, `dominant_*_ratio` 같은 요약 통계로만 사용한다.
